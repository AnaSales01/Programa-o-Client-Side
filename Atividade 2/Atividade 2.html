Linguagens

  1. CSS (Cascading Style Sheets)
    2.

    Criado em 1994 por Hakon Wium Lie, que queria facilitar a programação de sites que na época era muito mais complexa. As pessoas tinham que utilizar mais códigos para chegar a um resultado simples.
    O CSS serve para dá uma camada de personalização ao conteúdo visível.

  2. PHP (Personal Home Page)
    3.

    Foi criado em 1994 por Ramus Lerdof, a primeira versão era um simples conjunto binário Common Gateway Interface (CGI) escrito em linguagem de programação C. Originalmente usado para acompanhamento de visitas para o seu currículo online. Ao longo do tempo, mais funcionalidades foram desejadas, e Ramus reescreveu o PHP tools, produzindo uma maior e rica implementação. Este novo modelo foi capaz de interações com Banco de Dados e mais, fornecendo uma estrutura no qual os usuários poderiam desenvolver simples e dinâmicas interações web.

  3. HTML (Hypertext Markup Language)
    4.

    Foi criado em 1991, por Tim Bernes-Lee, no CERN na Suíça. Inicialmente o HTML foi projetado para interligar instituições de pesquisa próximas, e compartilhar documentos com facilidade. Em 1992, foi liberada a biblioteca de desenvolvimento WWW (world wide web), uma rede de alcance mundial, que junto com o HTML proporcionou o uso em escala mundial da WEB.

  4. TypeScript
    5.

    É um super conjunto tipado de JavaScript que é compilado para JavaScript puro. Ele segue o padão JavaScript e também é uma linguagem multi-paradigma, no entanto, adiciona funcionalidades e sintaxe de orientação ao objeto, classes e tipagem vistas em linguagens com C# e Java.

  5. JavaScript
    6.

    JavaScript é uma Linguagem de Programação, criada por Brendan Eich, a pedido da empresa Netscape, em meados de 1995. O JavaScript, a princípio, foi criado com o objetivo principal de validar formulários HTML. Na verdade, o JavaScript foi concebido para rodar no servidor. No entanto, com o passar do tempo isso foi modificado e, hoje, o JavaScript é uma linguagem Client-side.

Protocolos

  1. SFTP (Simple Transfer Protocol)
    2.

    consiste no protocolo FTP acrescido de uma camada de proteção para arquivos transferidos. Nele, a troca de informações é feita por meio de pacotes com a tecnologia SSH (Secure Shell – Bloqueio de Segurança), que autenticam e protegem a conexão entre cliente e servidor. O usuário define quantos arquivos serão transmitidos simultaneamente e define um sistema de senhas para reforçar a segurança.

  2. SSH (Secure Shell)
    3.

    é um dos protocolos específicos de segurança de troca de arquivos entre cliente e servidor. Funciona a partir de uma chave pública. Ela verifica e autentica se o servidor que o cliente deseja acessar é realmente legítimo.

  3. SSL (Secure Sockets Layer)
    4.

    permite a comunicação segura entre os lados cliente e servidor de uma aplicação web, por meio de uma confirmação da identidade de um servidor e a verificação do seu nível de confiança. Ele age como uma subcamada nos protocolos de comunicação na internet (TCP/IP).

  4. TCP/IP (Transmission Control Protocol/ Internet Protocol)
    5.

    Juntos, são os responsáveis pela base de envio e recebimento de dados por toda a internet.

  5. FTP (File Transfer Protocol)
    6.

    É a forma mais simples para transferir dados entre dois computadores utilizando a rede.
